# 15 · Профіль і налаштування

Екран «Профіль» (нижній таб «Налаштування») — це місце акаунта, мовних дефолтів, перемикачів і статистики. Документ описує кожен елемент: **що робить → на що впливає → стан**. Позначки: **[ЗАРАЗ]** (як у коді) / **[НОВЕ]** (затверджена зміна) / **[МАЙБУТНЄ]**.

Джерела коду:
- UI: `App.kt:4086–4630` — `ProfileScreen` і його під-компоненти (`ProfileSignInCard`, `ProfileIdentityCard`, `ProfileStat`, `SettingsGroup`, `SettingRow`, `Toggle`).
- Колбеки/вотиринг екрана: `App.kt:741–768`.
- Синк профілю на сервер: `App.kt:405–440` (`pushProfileSettings` → `updateCurrentUser`).
- Стор: `VocabeeStore.kt:149–157` (тоглі), `:378–406` (мовні селектори), `:449–455` (`signOutKeepLastUserState`).
- Бекенд: `vocabee-gateway/src/users/users.controller.ts` (`PATCH /me`), `users.service.ts:48–63` (`updateProfile`), `dto/update-profile.dto.ts`, `dto/user-response.dto.ts`.

Крос-лінки: онбординг/вхід — [02](02-onboarding-and-launch.md); економіка/баланс — [04](04-coins-economy.md); промо/«Запросити друзів» — [05](05-promo-api-and-banners.md); sync/мерж/вихід — [06](06-sync-and-account-merge.md); мови/мовлення/теми — [08](08-languages-speech-themes.md); крайові випадки (Help/Invite/нотифікації/тема) — [10](10-edge-cases-and-open-items.md); тренування/стрік — [11](11-practice-training.md).

## 3. «Мови за замовчуванням» (`App.kt:4157–4182`)

Два рядки `SettingRow` у групі + пояснювальний підпис «Нові словники створюються з цією парою мов автоматично.» (`App.kt:4174`).

| Рядок | Що робить | На що впливає | Стан |
|-------|-----------|---------------|------|
| **«Я розмовляю»** | Прапор + назва `userLanguage`; tap → `onSpeakingClick` → відкриває `LanguageForProfile(Speaking)` ботомшит (`App.kt:765`) | Дефолтна мова-джерело для **нових** словників (D6) | [ЗАРАЗ] |
| **«Я вивчаю»** | Прапор + назва `learningLanguage`; tap → `onLearningClick` → `LanguageForProfile(Learning)` (`App.kt:766`) | Дефолтна мова-ціль для **нових** словників (D6) | [ЗАРАЗ] |

**Вибір у ботомшиті** (`App.kt:843–864`):
- `onPick` → `SelectSpeakingLanguage` / `SelectLearningLanguage` → потім `pushProfileSettings()` (`App.kt:856–859`).
- Ботомшит **виключає** вже зайняту мову (`excludeCode`, `App.kt:851`) — UI-захист інваріанта «різні мови».

**Інваріант різних мов** (D6): на рівні стора, якщо нову мову збігли з протилежною, протилежну авто-перемикають на першу іншу підтримувану:
- `selectSpeakingLanguage` (`VocabeeStore.kt:378–391`): якщо `learningLanguage.code == language.code` → learning ставиться у `supportedLanguages.first { it.code != language.code }`.
- `selectLearningLanguage` (`VocabeeStore.kt:393–406`): симетрично для `userLanguage`.
- Обидва кличуть `persistLanguageChoice()` + `touchLocalRevision()`.

**Синк на сервер** (D6, D1): `pushProfileSettings()` (`App.kt:405–440`):
- Лише для `Authenticated` (`App.kt:407`); для аноніма — no-op (мови живуть локально, D2).
- Будує `LocalProfileSyncState` зі `speakLang/learnLang/notificationsEnabled/darkThemeEnabled` (`App.kt:408–413`).
- Викликає `backend.updateCurrentUser(UpdateProfileRequest(...))` → `PATCH /me` (`users.controller.ts:26–34`).
- Сервер пише в `users` (`users.service.ts:48–63`: `speakLang`, `learnLang`, `updatedAt=now()`), валідує мови через `@IsIn(SUPPORTED_LANGUAGE_CODES)` (`update-profile.dto.ts:13–21`), повертає `UserResponseDto`.
- Відповідь застосовується назад через `ApplyAuthenticatedAccount` (`App.kt:424–435`) — сервер як джерело істини.
- Помилка пушу мовчки ковтається; локальна ревізія лишається «брудною», ретрай на startup-sync (`App.kt:436–438`).

> **Існуючі словники не змінюються** при зміні дефолту (D6) — пара фіксується на момент створення словника. Деталі мов/STT/тем — [08](08-languages-speech-themes.md).

## 4. «Налаштування»: тоглі

### 4.1 Сповіщення (`App.kt:4186–4204`)

- **Що робить:** `SettingRow` з іконкою дзвоника, підпис «Нагадування про тренування», `Toggle(on = notificationsEnabled)`. Tap по рядку або тоглу → `onNotificationsChanged(!notificationsEnabled)`.
- **Колбек** (`App.kt:752–755`): `SetNotificationsEnabled(it)` → потім `pushProfileSettings()`.
- **Стор** (`VocabeeStore.kt:149–152`): оновлює `state.notificationsEnabled` + `touchLocalRevision()`. **Локально в prefs не зберігається** (на відміну від темної теми — див. 4.2); живе в state та синкається на сервер.
- **Бекенд:** колонка `notificationsEnabled` (`update-profile.dto.ts:23–26`, `users.service.ts:55`, `user-response.dto.ts:11`).
- **На що впливає:** **Припущення (уточнити)** — реального механізму пушів/локальних нотифікацій у коді **немає**; тогл лише зберігає булеве значення. Що саме має надсилатися: щоденне нагадування «час тренування» (під підпис рядка) — потребує підтвердження тригерів, частоти, каналу. Див. відкриті питання в [10](10-edge-cases-and-open-items.md).
- **Стан:** [ЗАРАЗ] лише прапорець (UI + збереження). [МАЙБУТНЄ] фактична доставка нагадувань про тренування ([11](11-practice-training.md)).

### 4.2 Темна тема (`App.kt:4206–4224`)

- **Що робить:** `SettingRow` з іконкою місяця, `Toggle(on = darkThemeEnabled)`. Tap → `onDarkThemeChanged(!darkThemeEnabled)`.
- **Колбек** (`App.kt:756–758`): `SetDarkThemeEnabled(it)` → `pushProfileSettings()`.
- **Стор** (`VocabeeStore.kt:153–157`): пише `preferencesManager.darkThemeEnabled` (зберігається локально, переживає рестарт), оновлює state, `touchLocalRevision()`.
- **Застосування теми:** `VocabeeTheme(darkTheme = state.darkThemeEnabled)` (`App.kt:292`) огортає весь застосунок → значення **реально перемикає** тему MaterialTheme.
- **Застереження (частково/візуально):** прототипний UI масово використовує **захардкоджені** `PrototypeColor.*` (напр. `PrototypeColor.Background` у `ProfileScreen`, `App.kt:4105`, і десятки інших місць). Тобто M3-тема перемикається, але прототипна палітра екранів — статична; темна тема покриває кольори частково. **Припущення (уточнити):** чи прототипні кольори мають реагувати на `darkThemeEnabled`.
- **На що впливає:** візуальна тема; синкається на сервер (`darkThemeEnabled`, `user-response.dto.ts:12`) для консистентності між пристроями.
- **Стан:** [ЗАРАЗ] перемикач працює для M3-теми + локальне/серверне збереження; повний дарк-режим прототипних кольорів — частковий. [МАЙБУТНЄ] токенізувати `PrototypeColor` під тему.

## 6. «Вийти» (`App.kt:4258–4280`)

- **Видимість:** лише `account is Authenticated` (`App.kt:4258`). Для аноніма кнопки немає.
- **Що робить** (колбек `App.kt:760–764`): чистить токени (`preferencesManager.accessToken = null`, `refreshToken = null`) → `store.signOutKeepLastUserState()`.
- **`signOutKeepLastUserState`** (`VocabeeStore.kt:449–455`): обнуляє `currentUserId`, переводить `account = Anonymous`, перезавантажує `topics` через `loadUserTopicsUseCase()`.
- **Ключове:** **`lastAuthenticatedUserId` НЕ очищається** при виході — він був записаний у `ApplyAuthenticatedAccount` (`VocabeeStore.kt:432`) і зберігається. Це дозволяє при повторному вході розпізнати «той самий користувач повертається» проти «новий email» для логіки sync/merge (D9) — див. [06](06-sync-and-account-merge.md).
- **Споріднено:** `clearAuthenticatedSessionForAnotherEmail` (`App.kt:442–449`) — окремий шлях «увійти іншим email» (теж кличе `signOutKeepLastUserState`, додатково чистить `currentUserId` і pending-конфлікт), вживається у sync-конфлікті D9.
- **На що впливає:** після виходу — анонімний режим (без монеток/реклами/промо, D2); локальні словники лишаються.
- **Стан:** [ЗАРАЗ].

## 7. Рядок статистики (`ProfileStat` ×3)

Три плитки `ProfileStat` у ряд — усі три [ЗАРАЗ] **реальні** (хардкод «7»/«12» прибрано):

| Плитка | Значення | Джерело |
|--------|----------|---------|
| 🔥 «днів поспіль» (стрік) | `state.streakDays` | `VocabeeStore.refreshedStreakDays()`: порівняння локальних півночей (`startOfDayEpochMillis`), персист у prefs (`streakDays` + `lastActiveDayStartMillis`). Активність сьогодні тримає стрік, «вчора» → +1, пропуск дня → скидання до 1. Округлення відстані в днях робить перехід DST безпечним. Оновлюється на старті апки і після кожного завершеного раунду |
| 🔖 «слів збережено» | `totalWords` = `topics.sumOf { it.words.size }` | обчислюване, як і раніше |
| 🃏 «тренувань» | `state.practiceRounds` | `VocabeeStore.recordPracticeRoundCompleted()` — інкремент у `moveNext` при `done=true` (класика **і** контекст), персист у prefs (`practiceRoundsCompleted`). Вихід із раунду хрестиком не рахується |

Підписи плиток відмінюються через `ukPlural(n, one, few, many)` (`App.kt`): «1 день / 2 дні / 5 днів», «1 слово / 2 слова / 5 слів», «1 тренування / 5 тренувань» — з поправкою на 11–14. Той самий хелпер застосовано в шапці головної («N слів зібрано», «N словників»).

> Лічильники локальні (SharedPreferences / NSUserDefaults), на сервер не синхронізуються. [МАЙБУТНЄ] серверний стрік/лічильник під D4-кампанії (daily_streak) і поля прогресу D10 — див. [11](11-practice-training.md).

## [НОВЕ] Запросити друзів + Допомога (design board 5)

Два пуш-екрани з Профілю (`EngagementScreens.kt`, роути `InviteFriends`/`HelpSupport`).

**Запросити друзів:** hero-картка (purple radial + honeycomb) із **справжнім QR**
(бібліотека `qrose`, лого в центрі) на реф-лінку `vocabee.app/i/<code>`; код генерує
gateway (`GET /v1/referral/me`, колонка `users.referral_code`, міграція 0008) зі слага
імені + суфікса. Рядок лінки з копіюванням (снекбар «Лінк скопійовано») і «Поділитися»
через системний share (`ShareController`: Android Intent / iOS UIActivityViewController).
Гість бачить загальну лінку + підказку увійти.
**[МАЙБУТНЄ]:** нарахування +10 обом після першого входу друга (deferred deep-link
атрибуція); швидкі кнопки месенджерів (Telegram/WhatsApp/Viber — лише встановлені).

**Допомога та підтримка:** чипи теми (bug/idea/billing/other), повідомлення ≥10 символів,
email — з акаунта автоматично (сервер підставляє з users) або обовʼязкове поле для гостя;
`POST /v1/support` (optional-auth) → таблиця `support_requests`; успіх = снекбар
«Надіслано! Відповімо протягом доби» + повернення в Профіль.
**[МАЙБУТНЄ]:** вкладення скріншотів (до 3 × 5MB, стискання 1080px) — потребує
завантаження файлів на gateway.
