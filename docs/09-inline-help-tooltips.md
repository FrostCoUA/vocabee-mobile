# 09 — Інлайн-довідка «?» (tooltip → bottom sheet)

> Повний інвентар інлайн-підказок «знак питання». Клік по іконці «?» відкриває нижню шторку (`PrototypeBottomSheet`) з коротким поясненням неочевидної концепції.
>
> **Статус усього файлу: [НОВЕ].** Зараз інлайн-довідки в застосунку **немає**. Єдиний слід «допомоги» — мертвий рядок «Допомога та підтримка» у профілі з порожнім `onClick = {}` (`App.kt:4252` / `App.kt:4254`), який нічого не робить. Цей файл описує затверджений новий функціонал.

---

## 1. Навіщо це потрібно

Vocabee має кілька неочевидних механік (монетки, ліміти аноніма, напрямок STT, sync-конфлікт, мерж), які не вміщаються в підписи елементів і не повинні захаращувати головний UI. Замість довгих текстів біля кожного контрола ставимо компактну іконку «?», що по кліку відкриває коротке пояснення. Це знижує поріг входу й прибирає здогадки користувача «чому з мене зняли монетку».

Принципи:
- **Коротко.** 2–4 речення на шторку, людською мовою, без жаргону.
- **Контекстно.** «?» стоїть рядом саме з тим елементом, який пояснює.
- **Без дублювання логіки.** Текст пояснень — це контент, а не бізнес-правила. Повна механіка промо/економіки живе у файлах `05-*` (промо) та документації економіки.
- **Опційна дія.** Якщо доречно, шторка має кнопку-CTA («Дивитись рекламу», «Увійти через Google»), що веде до відповідного флоу.

## 2. UX-патерн «?»

### Вигляд іконки
- Маленька кругла іконка зі знаком питання (`PrototypeIcon.Help` — той самий, що вже використовується в рядку профілю, `App.kt:4246`), розмір ~16–18 dp, колір `PrototypeColor.Muted` / `Muted2` (приглушений, щоб не перетягувати увагу з основного контента).
- Клікабельна зона ≥ 40 dp (хіт-таргет більший за візуальну іконку).
- На головних інтерактивних елементах «?» НЕ ставимо — лише там, де концепція справді неочевидна (див. інвентар нижче).

### Де ставити «?»
- **Біля заголовків секцій** — напр. «Гаманець», «Гостьовий режим», «Акції», «Синхронізація».
- **Поряд із цінниками / лімітами** — біля «−10 монеток», «−1 монетка», «2/2 словники», «50 слів».
- **Поряд із перемикачами / контролами** — напр. перемикач напрямку перекладу (STT), пара мов у налаштуваннях, темна тема, нотифікації, тема (іконка/колір) словника.

### Як відкривається
- Клік → `PrototypeBottomSheet` (`PrototypeBottomSheets.kt:60`). Це той самий компонент модальної шторки, що вже використовується для створення словника та вибору мови, тож стиль уніфікований (заголовок зліва, кнопка «Закрити» справа `PrototypeBottomSheets.kt:101-115`, drag-handle зверху).

### Що всередині шторки
1. **Заголовок** (`title` шторки) — назва концепції.
2. **Текст пояснення** — 2–4 речення (готовий копірайт у таблиці нижче).
3. **Опційна кнопка дії** (`foot = { ... }` слот шторки, `PrototypeBottomSheets.kt:121-124`) — напр. `PrimaryPillButton` «Дивитись рекламу» / «Увійти через Google». Кнопку показуємо лише там, де є природна наступна дія; інакше шторка чисто інформаційна (без `foot`).

## 5. Залежності та посилання

- Повна механіка акцій (стрік / milestone / leaderboard / one_time, config-driven промо API) — див. файл `05-*` (промо). Тут лише короткі пояснення для шторок #5–#8.
- Економіка та авторитетність сервера (D1), анонім без монеток (D2) — копірайт #1–#4, #9, #10, #20 має узгоджуватися з рішеннями D1/D2.
- Видалення без повернення монеток + Undo (D3) — копірайт #16.
- Sync-конфлікт / мерж при вході (D9) — копірайт #15.
- Напрямок STT по словнику (D8) — копірайт #11.
- Мова словника vs профілю (D6) — копірайт #12.
- Теми/іконки словників (D7) — копірайт #13.

**Припущення (уточнити):**
- Точні екрани для секцій «Акції», «Синхронізація», «Сповіщення», «Темна тема» та перемикача напрямку STT ще не реалізовані в коді ([НОВЕ]), тому конкретні `file.kt:рядок` для тригерів #5–#8, #11, #14, #17–#19 будуть визначені при імплементації відповідних екранів.
- Чи робити рядок «Допомога та підтримка» (`App.kt:4252`) точкою входу до повного списку довідок, чи лишити суто інлайн-«?» — рішення за продуктом.
