# 12 — Motion & Interaction Brief (для апгрейду дизайну)

> Призначення: детальний опис флоу й анімацій, який згодовується дизайн-інструменту, щоб **апгрейднути** раніше намальовані макети (банери, промо-шит, мерж, онбординг, створення словника, тултіп, лідерборд, картка тренування + підсумок).
> Позначки: **[ЗАРАЗ]** — як працює в коді сьогодні, **[ЦІЛЬ]** — апгрейд, який треба намалювати/закласти.

---

## 0. Глобальні токени руху (застосовувати всюди)

| Токен | Значення | Де |
|---|---|---|
| Easing «standard» | `cubic-bezier(0.2, 0, 0, 1)` (ease-out-ish) | входи, перевороти, settle |
| Easing «emphasized» | `cubic-bezier(0.2, 0, 0, 1)` з overshoot/spring (stiffness ~ medium, damping ~ 0.75) | поява призів, claimable, «дзиґа» |
| Тривалість S | 120–180 мс | мікро (натиск, tick, морф кнопки) |
| Тривалість M | 260–320 мс | поява/зникнення, reflow списку, заповнення прогресу |
| Тривалість L | 420–460 мс | переворот картки, морф-оверлей додавання слова |
| Тінь spec | flat: без розмиття під час стрімінгу; elevation лише функціональна | картки, шити |
| Haptic «light» | tick | натиск, поріг свайпу, «Знаю» |
| Haptic «medium» | bump | «Не знаю», коміт видалення |
| Палітра | Purple `#4F46E5` / Deep `#410FA3` / Tint `#E0E7FF`; Yellow `#FFCC00`; Orange `#F76400` / Peach `#FFE8D6`; Green `#15803D` / `#DCFCE7`; Red `#DC2626`; Ink `#111827`; Muted `#6B7280`; Line `#E5E7EB` | бренд Vocabee |

Принцип: рух **підказує наслідок** (куди поділось слово, що змінилось у прогресі), а не прикрашає. Кожна руйнівна дія має **зворотний хід** (undo або контр-анімація.)

## A. Картка тренування — переворот (flip)

### A.1 Анатомія
- **Контейнер сесії [ЗАРАЗ]:** running і done fullscreen, без нижнього таб-бару. Збільшена картка займає весь доступний простір; у шапці праворуч від «Тренування» — хрестик виходу.
- **Передня грань (питання):** біла поверхня, рамка 2dp кольору теми (`card.accent`), радіус 28dp. Усередині: чип теми (крапка + назва), велике вихідне слово (40sp, ExtraBold), IPA, рівень/заливка знання (`KnowledgeBackgroundFill`), окрема кнопка TTS і внизу контекстне речення конкретного sense (якщо є). Цільове слово/словоформа підсвічується жовтим через той самий `highlightedSentence`, що й у context-mode.
- **Задня грань (відповідь):** поверхня кольору `card.accent` (залита), переклад білим (32sp), без рамки.
- **Вихід [ЗАРАЗ]:** хрестик або системний Back під час незавершеної сесії відкриває confirmation bottom sheet; «Продовжити» закриває шторку без зміни позиції, «Перервати» повертає на setup.

### A.2 [ЗАРАЗ] Поточна анімація
- Кожен flip додає до цільового `rotationY` півоберт: tap/right `+180°`, left `−180°`; `tween(460ms)`, `cameraDistance = 14 × density` (перспектива 3D).
- Видима грань обчислюється за фактичним кутом, нормалізованим у `0°..360°`: back лише в діапазоні `90°..270°`. Тому при зворотному русі поточна сторона не підміняється до проходження ребра.
- Контент задньої грані контр-обертається `rotationY = 180°`, щоб текст не був дзеркальним.
- Тригер: тап або горизонтальний swipe/fling по поверхні картки. Жест **лише** перемикає грань і не змінює індекс/оцінку. Тап по вкладеній кнопці динаміка лише запускає TTS і не передається картці.

### A.3 [ЦІЛЬ] Апгрейд перевороту
- Замінити лінійний `tween(460)` на **spring** (emphasized): легкий overshoot ~3–4° на завершенні — картка «доводиться» пружно.
- Додати **підйом**: під час перевороту картка масштабується `scale 1.0 → 1.04 → 1.0` і тінь трохи росте (фокус на активній картці).
- **Зміна грані** — не різко на 90°, а з мікро-кросфейдом контенту в вікні 80–100° (fade 80ms), щоб не «блимало».
- **Idle-підказка** до першого тапу: дуже легке погойдування картки `rotationY ±4°` раз на ~3с (loop), що натякає «торкнись».
- **Haptic light** у момент перетину 90° (фіксація грані).
- Глибина: збільшити перспективу (менший cameraDistance) для відчутнішого 3D, але без «ламання» тексту.

## B. Флов «Не знаю» (та порівняння зі «Знаю»)

### B.1 [ЗАРАЗ] Поточний флов
Стан: показана **передня** грань (слово), внизу дві кнопки — «Не знаю» (peach/orange) і «Знаю» (green).
- Тап **«Не знаю»** → `knowledgePercent −20%`; картка **авто-перевертається** на відповідь (`flipped=true`); ряд із двох кнопок **замінюється однією** кнопкою «Далі»; стан `waitingForNextAfterMiss=true` (поки не натиснеш «Далі», далі не йде; тап по картці перемикає грані).
- **Ручний tap/swipe/fling для відкриття відповіді** має той самий семантичний наслідок, що «Не знаю»: `−20%` застосовується один раз, кнопки одразу змінюються на «Далі». Зворотний flip не повертає варіант «Знаю».
- Колода — `HorizontalPager` з окремим flip-станом на сторінку; «Знаю»/«Далі» запускають горизонтальний slide (~420 мс), а нова сторінка завжди входить передньою гранню. Ручне pager-гортання вимкнене, щоб не конфліктувати з directional flip і не обходити оцінку.
- Тап **«Далі»** → наступна картка (`flipped=false`, `index++`).
- Іконка-стрілка в кнопці **«Далі»** стоїть праворуч від тексту.
- Тап **«Знаю»** → `+20%`, `correctAnswers++`, **одразу** наступна картка (без показу відповіді).

### B.2 [ЦІЛЬ] Апгрейд флоу «Не знаю» — покроково з анімаціями
1. **Натиск «Не знаю»** (0мс): кнопка дає `scale 0.98` + haptic medium; по картці пробігає **червоний пульс** (рамка/обводка спалахує `#DC2626` alpha 0.0→0.25→0.0, 200мс) — візуальний сигнал «помилка».
2. **Авто-переворот на відповідь** (S→L): той самий flip із A.3, але ініційований системою (не тапом).
3. **Кільце знання вниз** (M): `knowledgePercent` анімовано спадає `−20%` (заливка картки опускається 280мс ease-out, цифра % count-down).
4. **Морф кнопок** (S): дві кнопки **зливаються в одну** «Далі» — «Знаю» згортається до 0 ширини й тане, «Не знаю» розтягується на всю ширину й перефарбовується у нейтрально-фіолетову. 160мс.
5. **[ЦІЛЬ нове] Перечергування** (doc 11): слово, яке не знав, **повертається в кінець колоди** цієї ж сесії (а не зникає). У підсумку показуємо, що його ще раз спитають.
6. **Тап «Далі»** → **вихідний перехід картки**: поточна картка йде вліво `translateX -100% + fade` (M), наступна входить справа `translateX +12% → 0 + fade` (M, з невеликим стагером). Прогрес-бар крокує (див. C).

### B.3 «Знаю» — апгрейд (для контрасту)
- Натиск → `scale 0.98` + haptic light; **зелена галочка-спалах** у центрі картки (scale-in 0→1, fade-out, 320мс).
- Картка **не** перевертається — одразу вихідний перехід (вліво) + наступна входить.
- Кільце знання **вгору** `+20%` анімовано (заповнення піднімається).
- `correctAnswers` count-up.

> Сенс контрасту: «Знаю» — швидко, легко, зелено, без показу відповіді; «Не знаю» — змушує **побачити відповідь** і свідомо натиснути «Далі» (червоний акцент, медіум-haptic, перечергування).

## C. Прогрес сесії

### C.1 [ЗАРАЗ]
- Текст-лічильник: `"{index+1} / {deck.size} · правильно {correctAnswers}"`.
- Прогрес-бар: ширина = `(index+1)/deck.size`.
- Кільце/заливка знання — на самій картці.

### C.2 [ЦІЛЬ] Апгрейд прогресу
- **Анімоване заповнення** бару при переході картки: ширина росте `tween(300, ease-out)`, а не стрибком.
- **Сегментований бар** (10 поділок) — кожна картка = сегмент; пройдені заповнюються, поточна підсвічена. Зрозуміліше «скільки лишилось».
- **Count-up** «правильно N» при «Знаю» (швидкий roll цифри 200мс).
- **Колірний натяк**: сегмент відповіді фарбується зелено (Знаю) / помаранчево (Не знаю) — наприкінці видно «мапу» сесії.
- **Кінець колоди**: бар доходить до 100% → коротка пауза (150мс) → **перехід у Done**: картки розлітаються/тануть, екран підсумку входить (scale-in 0.96→1 + fade, M). Конфеті-іконка робить spring pop.
- На картці: кільце знання оновлюється плавно (див. B), а не миттєво.

## D. Свайп-видалення СЛОВА (рядок у словнику)

### D.1 [ЗАРАЗ] Поточна механіка (reveal-патерн)
`SwipeRevealDeleteContainer`, кнопка ширини **88dp**:
- Свайп вліво **відкриває** червону панель «Видалити» (іконка ✕ + підпис) під рядком; рядок зсувається `translationX`, clamp `[-88dp, 0]`.
- Відпускання → **settle** `tween(170ms)`: якщо швидкість `< -320` → відкрити повністю; `> 320` → закрити; інакше поріг `42%` ширини → відкрити/закрити.
- Тап по відкритій кнопці → рядок повертається в 0 і слово **видаляється одразу** (без підтвердження, без undo).
- Видалення групи слів (`group.translations.forEach(onRemoveWord)`) прибирає всі переклади групи.

### D.2 [ЦІЛЬ] Апгрейд свайп-видалення слова
1. **Прогресивний реквіл** (під час драгу): іконка ✕ у червоній панелі **масштабується** від 0.6→1.0 у міру відкриття; панель трохи темнішає ближче до порогу.
2. **Поріг-перетин** (на `42%` або новому `~50%`): haptic light + іконка робить мікро-pop (scale 1.1) — тактильно зрозуміло «зараз спрацює».
3. **[ЦІЛЬ нове] Full-swipe-to-delete**: якщо протягнути далеко (`> ~65%` ширини або сильна швидкість), **не зупинятись на кнопці**, а одразу комітити видалення (як у Gmail/Telegram). Червона панель розширюється на всю ширину, рядок вилітає вліво.
4. **Колапс рядка** (M): після коміту рядок **схлопується по висоті** `height → 0` + `fade`, сусіди **плавно під'їжджають** (list reflow, `tween(300, ease-out)`, бажано spring для природності).
5. **[ЦІЛЬ нове за D3] Undo-снекбар**: одразу після видалення знизу спливає снекбар «Слово видалено · Скасувати» на **6с**. Тап «Скасувати» → рядок **повертається** зворотною анімацією (висота 0→авто + fade-in, сусіди розсуваються). Якщо снекбар згас — видалення фіналізується й синкається.
6. **Резистанс**: за межею максимального відкриття — легкий rubber-band (затухаюче розтягнення на 8–10px).
7. Слова видаляємо **без діалогу-підтвердження** (швидко) — безпеку дає Undo, а не модалка.

## E. Свайп-видалення СЛОВНИКА (картка на головній)

### E.1 [ЗАРАЗ]
`SwipeRevealDeleteContainer`, кнопка **76dp** на `DictionaryCard`:
- Свайп вліво відкриває «Видалити» → тап → відкривається **шторка підтвердження** `DeleteDictionaryConfirmationSheet` («Видалити словник? І N слів буде видалено. Цю дію не можна скасувати.»).
- Підтвердження → словник видаляється (каскадно зі словами); якщо ти був усередині нього — повертає назад.

### E.2 [ЦІЛЬ] Апгрейд видалення словника
1. **Зберігаємо підтвердження-шторку** (бо видалення **каскадне** — губляться всі слова): це руйнівніше за одне слово, тож модалка доречна. Але апгрейднути її появу: шторка входить знизу spring (M), затемнення тла fade 0→0.45.
2. **Коміт** → картка на головній **згортається**: `scale 1→0.92 + fade` (S→M), а сітка словників **переукладається** — сусідні картки під'їжджають на місце (list reflow, `tween(300)`/spring). Не різкий зник, а «втягування».
3. **[ЦІЛЬ за D3] Undo-снекбар** і тут: «Словник видалено · Скасувати» 6с (поряд із підтвердженням — підтвердження проти випадкового свайпу, undo проти «передумав»). Скасування відновлює картку зворотним рухом.
4. **Текст про монетки**: якщо словник був платний (3-й+), у шторці/снекбарі чесно показати «Монетки за створення не повертаються» (D3). Без рефанду.
5. **Альтернативний вхід**: довге утримання картки → контекст-меню (Перейменувати/Змінити іконку/Видалити) — як другий, обережніший шлях окрім свайпу.
6. **Свайп-реквіл картки**: під час драгу сама картка трохи **нахиляється/масштабується** (scale 0.98), червона зона з іконкою кошика проявляється з прогресивним масштабом іконки (як у D.2).

### E.3 Різниця слово vs словник (закласти в дизайн)
| | Слово | Словник |
|---|---|---|
| Свайп | reveal → (ціль) full-swipe коміт | reveal → тап |
| Підтвердження | **немає** (швидко) | **шторка** (каскад) |
| Захист | **Undo-снекбар 6с** | **Шторка + Undo-снекбар 6с** |
| Анімація прибирання | колапс висоти + reflow списку | згортання картки + reflow сітки |
| Монетки | — | без рефанду, повідомити |

## F. Дрібні взаємодії, що варто апгрейднути заразом
- **Додавання слова** (морф-оверлей із пігулки, `tween ~420мс`) — лишити, але узгодити easing зі spring-набором; «+» на рядку результату → коротка пульсація + галочка (вже задумано) + слово «падає» в словник.
- **FAB «+»** на головній: натиск scale 0.94 + haptic light; поява нового словника згори сітки — вхід card із scale-in.
- **Перемикач напрямку мов** у словнику (впливає на STT, doc 08): перемикання — горизонтальний swap-флип двох мовних пігулок (rotateY/translate, S), із мікро-haptic; біля нього «?» → тултіп-шит (doc 09).
- **Банери/промо** (doc 05): `claimable`-стан — пульсація бейджа винагороди (scale 1.0↔1.06, 1.2с loop, emphasized) + легке світіння рамки; перехід `active→claimable` — короткий «success»-сплеск.

## G. Підсумок для дизайн-інструменту (що саме намалювати/оновити)
1. Картка тренування: spring-flip із підйомом, idle-погойдування, кросфейд грані, 3D-перспектива.
2. Флов «Не знаю»: червоний пульс → авто-flip → морф двох кнопок в одну «Далі» → перечергування → вихід картки вліво/вхід наступної. Контраст зі «Знаю» (зелена галочка, без показу відповіді).
3. Прогрес: сегментований бар (10), анімоване заповнення, count-up «правильно», колірна мапа сесії, перехід у Done зі spring-pop.
4. Свайп слова: прогресивний реквіл + full-swipe коміт + колапс висоти + reflow + Undo-снекбар 6с (без модалки).
5. Свайп словника: реквіл + шторка-підтвердження (каскад) + згортання картки + reflow сітки + Undo-снекбар + примітка про монетки; довге утримання як альтернатива.
6. Глобально: застосувати токени руху §0 (easing/тривалості/haptics), кожна руйнівна дія має зворотний хід.
